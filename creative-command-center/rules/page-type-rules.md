## SECTION 1: HOMEPAGE RULES

### Homepage Purpose

```
The homepage is NOT:
- A summary of everything
- A list of latest posts
- A feature showcase

The homepage IS:
- The first impression
- The brand statement
- The invitation to explore
- The trust builder
```

### Homepage Required Elements

```
MUST HAVE:
1. Clear brand identity (within 3 seconds)
2. Primary navigation
3. One clear entry point for new visitors
4. Value proposition (what's in it for me?)
5. Trust signal (social proof, credentials, or quality indicator)

MUST NOT HAVE:
- Multiple competing CTAs above fold
- "Latest posts" as primary content
- Generic stock imagery
- Every category crammed in
```

### Homepage Structure Rules

**Hero Section:**
```
PURPOSE: Emotional entry + single clear direction
DURATION: Full viewport or close to it
CONTENT: 
  - L1 headline (one)
  - Supporting text (brief)
  - Single primary CTA
FORBIDDEN:
  - Multiple CTAs competing
  - Wall of text
  - Generic tagline
  - Autoplay video with sound
```

**Section 2 (After Hero):**
```
PURPOSE: Validate the promise OR guide to value
OPTIONS:
  - Value proposition expansion
  - Featured/best content
  - "Start here" pathway
  - Social proof (NOT generic testimonials)
FORBIDDEN:
  - Immediately dumping into blog grid
  - 3-column feature repetition from template
```

**Section 3-4:**
```
PURPOSE: Deeper exploration invitation
OPTIONS:
  - Category/topic exploration
  - Content highlights
  - Trust building
FORBIDDEN:
  - Same layout as Section 2
  - Repetitive patterns
```

**Final Section (Before Footer):**
```
PURPOSE: Conversion opportunity
OPTIONS:
  - Newsletter signup
  - "What to do next" guidance
  - Community invitation
FORBIDDEN:
  - Aggressive popup instead
  - Identical to other CTAs on page
```

## SECTION 2: SINGLE POST/ARTICLE RULES

### Article Purpose

```
The article page IS:
- Long-form content consumption
- Deep value delivery
- SEO landing page
- Trust builder

The article page is NOT:
- Distraction zone
- Ad placement platform
- Popup territory
```

### Article Structure Rules

**Article Header:**
```
REQUIRED:
  - H1 (the title)
  - Publication date
  - Reading time estimate
  - Author (if applicable)
  - Category breadcrumb
  
OPTIONAL:
  - Featured image
  - Last updated date
  - Social share (subtle)

FORBIDDEN:
  - Multiple H1s
  - Sharing buttons taking over
  - Date in unclear format
```

**Article Body:**
```
LAYOUT:
  - Single column preferred (max 700-800px)
  - Or sidebar (if site uses sidebars consistently)
  
SPACING:
  - Generous paragraph spacing
  - Subheadings break up content
  - Images/media provide breathing room

TYPOGRAPHY:
  - 18px+ body text
  - 1.6-1.8 line height
  - Clear heading hierarchy

FORBIDDEN:
  - Sidebar competing with content
  - Ads interrupting flow
  - Excessive inline CTAs
  - Pop-ups while reading
```

**Table of Contents:**
```
WHEN: Articles > 2000 words
POSITION: After intro, before main content OR sticky sidebar
STYLE: Matches site aesthetic
BEHAVIOR: Smooth scroll to sections
```

**Article Footer (After Content):**
```
REQUIRED:
  - Author box (builds trust)
  - Related content (3-5 articles)
  - Comment section OR newsletter CTA

FORBIDDEN:
  - 20 "related posts" in grid
  - Multiple different CTAs
  - Aggressive exit intent
```

### Article Sidebar Rules (If Used)

```
IF sidebar exists:

CONTENT:
  - Newsletter signup
  - Popular posts (max 5)
  - About/author widget
  - Search

FORBIDDEN:
  - More than 4 widgets
  - Ads dominating
  - Same content as footer
  - Competing CTAs
```

## SECTION 3: CATEGORY/ARCHIVE RULES

### Archive Purpose

```
The archive page IS:
- Content discovery
- Topic exploration
- SEO landing page for topics

The archive page is NOT:
- Endless identical grid
- Blog dump
- SEO keyword stuffing
```

### Archive Structure Rules

**Archive Header:**
```
REQUIRED:
  - Category/archive title
  - Brief description (unique per category)
  - Post count (optional)

FORBIDDEN:
  - Generic "Posts in [Category]"
  - Duplicate descriptions across categories
  - No description at all
```

**Archive Content Display:**
```
OPTIONS (site decides in DNA):
  1. Featured + Grid
     - Top 1-3 posts featured larger
     - Remaining in grid
  
  2. Magazine Layout
     - Varied card sizes
     - Editorial feel
  
  3. List Layout
     - Clean list with excerpts
     - Scannable
     
  4. Card Grid
     - Uniform cards
     - (Most common but least interesting)

FORBIDDEN:
  - Identical cards with no hierarchy
  - Endless scroll without pagination
  - No indication of content depth
```

**Archive Pagination:**
```
OPTIONS:
  - Numbered pagination (traditional)
  - Load More button (better UX)
  - Infinite scroll (careful - can hurt UX)

FORBIDDEN:
  - No pagination (dump everything)
  - Tiny page sizes (5 posts per page)
  - Huge page sizes (50 posts causing slow load)
```

### Category Page Customization

**Each category can have:**
```
- Custom header image/color
- Unique description
- Featured posts selection
- Subcategory breakdown
- Category-specific sidebar content
```

**Per DNA, categories might:**
```
- Have completely different layouts
- Feature different content types
- Have different CTAs
```

## SECTION 4: ABOUT PAGE RULES

### About Page Purpose

```
The about page IS:
- Trust builder
- Connection point
- Brand story
- Credibility signal

The about page is NOT:
- Resume
- Mission statement dump
- Generic "about us" template
```

### About Page Required Elements

```
MUST HAVE:
1. Clear answer to "Who is behind this?"
2. Why this site exists
3. What makes it different/trustworthy
4. Human element (name, photo, personality)
5. Clear next step

MUST NOT HAVE:
- "We are passionate about..." (generic)
- Faceless corporation speak
- Just a wall of text
- No photos/personality
```

### About Page Structure

**Personal Brand Sites:**
```
- Lead with person, not product
- Story of how site came to be
- Credentials that matter
- Clear personality/voice
- Photo (real, not stock)
- Contact info or social links
```

**Multi-Author Sites:**
```
- Site story/mission first
- Team showcase
- Individual author pages (linked)
- Expertise demonstration
```

**Site-Specific About Approaches:**

```
WitchcraftForBeginners:
  - Personal story of coming to practice
  - Why this site is different (not fluffy)
  - Approach to teaching
  - Credentials (years of practice, training)
  
SmartHomeWizards:
  - Technical background
  - Testing methodology
  - Editorial standards
  - Device experience
  
Family-Flourish:
  - Personal family story
  - Why this approach
  - Qualifications if any
  - Real family photos
```

## SECTION 5: CONTACT PAGE RULES

### Contact Page Purpose

```
The contact page IS:
- Simple path to reach you
- Trust signal (real person behind site)
- Expectation setter

The contact page is NOT:
- Form with 20 fields
- FAQ dump
- Alternative sitemap
```

### Contact Page Required Elements

```
MUST HAVE:
1. Contact form (functional)
2. Response time expectation
3. Alternative contact (if applicable)

NICE TO HAVE:
- Brief intro text
- FAQ link
- Social links

MUST NOT HAVE:
- Required fields for non-essential info
- Captcha that blocks humans
- No confirmation after submit
- "We'll get back to you soon" (vague)
```

### Contact Form Rules

```
REQUIRED FIELDS (minimal):
- Name
- Email
- Message

OPTIONAL FIELDS:
- Subject/Topic dropdown
- How did you find us
- Urgency level

FORBIDDEN:
- Phone required
- Address required
- Company required (for personal blogs)
- Too many required fields
```

## SECTION 6: SEARCH RESULTS PAGE

### Search Results Purpose

```
The search page IS:
- Quick path to content
- Recovery from failed navigation
- Discovery tool

The search page is NOT:
- Generic list dump
- Poorly formatted results
- Dead end
```

### Search Results Requirements

```
MUST HAVE:
1. Clear query display ("Results for: [query]")
2. Result count
3. Relevant results first
4. Link to content

MUST NOT HAVE:
- No results found (with no guidance)
- Overwhelming number of fields
- Slow results
```

### No Results Handling

```
REQUIRED:
- Clear "no results" message
- Suggestions for alternative searches
- Link to browse categories
- Search tips

FORBIDDEN:
- Just "No results"
- Blank page
- Error message style
```

## SECTION 7: ERROR PAGES (404, 500)

### Error Page Purpose

```
Error pages ARE:
- Recovery opportunity
- Brand touchpoint
- Navigation assistance

Error pages are NOT:
- Dead ends
- Blank screens
- Generic server messages
```

### 404 Page Requirements

```
MUST HAVE:
1. Clear this is an error (but not scary)
2. Search functionality
3. Navigation to popular content
4. Link to homepage

BRAND OPPORTUNITY:
- Can be personality-rich
- Can include humor (if fits brand)
- Can include illustration
- Site-specific messaging
```

**404 Page Examples:**

```
WitchcraftForBeginners:
  Headline: "This path has faded into the mist..."
  Body: "The page you're looking for may have been moved or no longer exists."
  CTA: [Search] [Return to Homepage] [Browse Spells]
  
SmartHomeWizards:
  Headline: "Connection lost with this page"
  Body: "The page you're looking for isn't responding. It may have been moved."
  CTA: [Search] [Back to Home] [Browse Guides]

Family-Flourish:
  Headline: "Oops! This page wandered off..."
  Body: "We couldn't find what you were looking for, but we can help!"
  CTA: [Search] [Go Home] [Browse Topics]
```

### 500 Page Requirements

```
MUST HAVE:
1. Clear something is wrong (not user's fault)
2. Apology
3. What to try (refresh, wait)
4. Alternative navigation

MUST NOT HAVE:
- Technical jargon
- Stack traces
- Blame on user
```

## SECTION 8: LANDING PAGE RULES

### Landing Page Purpose

```
Landing pages ARE:
- Conversion-focused
- Single-purpose
- Traffic destination

Landing pages are NOT:
- Navigation hubs
- Content dumps
- Homepage clones
```

### Landing Page Structure

```
REQUIRED:
1. Clear value proposition (above fold)
2. Social proof
3. Feature/benefit explanation
4. Single CTA (repeated appropriately)
5. Objection handling

FORBIDDEN:
- Full navigation (distracting)
- Multiple different CTAs
- Competing offers
- Exit popup immediately
```

### Landing Page Types

**Lead Magnet Landing:**
```
Structure:
1. Headline (benefit-focused)
2. What they get
3. Why it's valuable
4. Simple form (name + email)
5. Trust signal

Forbidden:
- Long forms
- Multiple lead magnets
- Distracting navigation
```

**Product/Affiliate Landing:**
```
Structure:
1. Product intro
2. Key benefits
3. Feature breakdown
4. Social proof/reviews
5. Comparison (if relevant)
6. CTA

Forbidden:
- Hiding affiliate disclosure
- False urgency
- Competitor bashing
```

## SECTION 9: AUTHOR PAGE RULES

### Author Page Purpose

```
Author pages ARE:
- Trust builder
- E-E-A-T signal
- Content hub for that author

Author pages are NOT:
- Simple bio + list
- Hidden/inaccessible
- Generic template
```

### Author Page Requirements

```
MUST HAVE:
1. Author name and photo
2. Bio/credentials
3. Posts by this author
4. Contact/social (optional)

NICE TO HAVE:
- Areas of expertise
- Recent posts
- Total post count
- Featured posts by author
```

## SECTION 10: LEGAL PAGES RULES

### Legal Page Purpose

```
Legal pages (Privacy, Terms, etc.) ARE:
- Required compliance
- Trust signals
- User protection

Legal pages are NOT:
- Design showcases
- Brand opportunities (mostly)
```

### Legal Page Requirements

```
MUST HAVE:
1. Accessible from every page (footer)
2. Readable format (not wall of text)
3. Last updated date
4. Accurate information

NICE TO HAVE:
- Table of contents
- Plain language summaries
- Section headers
```
