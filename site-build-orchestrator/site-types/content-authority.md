## E-E-A-T Requirements (Critical)

### Experience
```
Show you've DONE this:
- Personal anecdotes in content
- "I've been practicing for X years"
- Real results/examples
- Mistakes made and learned from
- Photo/video of actual practice (where appropriate)
```

### Expertise
```
Demonstrate deep knowledge:
- Comprehensive, accurate information
- Proper terminology
- Source citations where applicable
- Nuanced understanding
- Address edge cases and variations
```

### Authoritativeness
```
Build recognition:
- Consistent author profiles
- Social proof (comments, shares)
- Media mentions
- Guest contributions
- Community engagement
```

### Trustworthiness
```
Create safety:
- Clear about limitations
- "Consult a professional" where needed
- No harmful advice
- Transparent about monetization
- Accurate, fact-checked content
```

## Homepage Structure

### Hero Section (Authority Sites)
```
Purpose: Create ATMOSPHERE + establish expertise
Elements:
- Evocative headline (not transactional)
- Welcoming subhead
- Single, inviting CTA
- Visual that sets tone

Example for WitchcraftForBeginners:
"Where Ancient Wisdom Meets Modern Seekers"
"Your trusted guide into authentic witchcraft practice."
CTA: "Begin Your Journey"

Example for MythicalArchives:
"Explore the Stories That Shaped Humanity"
"Scholarly mythology encyclopedia for curious minds."
CTA: "Enter the Archives"
```

```
Purpose: Clear path for beginners (main audience)

- "New Here?" prominent

```

### Section 2: Featured/Best Content
```

- Most comprehensive guides
- Reader favorites
```

### Section 3: Topic Exploration
```
Purpose: Show breadth of coverage

Content:
- Visual category navigation
- Quick descriptions
```

### Section 4: Community/Newsletter
```
Purpose: Build relationship
Layout: Warm CTA section
Content:
- Newsletter signup
- Community invitation
- "Join us" language
```

## Pillar Page Template

### Structure for Ultimate Guides
```
Target: 5,000-10,000 words
Purpose: Rank for competitive head terms
SEO: Target "what is [topic]", "[topic] guide"

Structure:
[Epic H1 Title]
[Author + Last Updated + Reading Time]
[Table of Contents - clickable]

1. Introduction (hook + what you'll learn)
2. What is [Topic] (definitive answer)
3. History/Background (context)
4. Core Concepts (the fundamentals)
5. How to [Action] (practical application)
6. Advanced Topics (for return visitors)
7. Common Mistakes (helpful warnings)
8. FAQ (schema opportunity)
9. Next Steps (internal links)
10. Resources (external authority links)

Internal Links: 15-30 to cluster content
External Links: 5-10 to authoritative sources
```

### Pillar-Cluster Model
```
Example for WitchcraftForBeginners:

PILLAR: "The Complete Beginner's Guide to Witchcraft"
├── CLUSTER: "What is Witchcraft? History & Traditions"
├── CLUSTER: "Setting Up Your First Altar"
├── CLUSTER: "Essential Witchcraft Tools"
├── CLUSTER: "Your First Simple Spell"
├── CLUSTER: "Understanding Magical Correspondences"
├── CLUSTER: "Witchcraft Ethics & Responsibility"
└── CLUSTER: "Common Beginner Mistakes"

All cluster posts link to pillar.
Pillar links to all clusters.
Clusters interlink where relevant.
```

## Content Quality Standards

### Minimum Requirements
```
Word Count:
- Pillar content: 5,000-10,000
- Standard posts: 2,000-3,500
- Quick reference: 1,000-1,500

Research Depth:
- Multiple sources consulted
- Cross-referenced information
- Scholarly sources where applicable

Originality:
- Unique insights and perspective
- Personal experience included
- Not rehashed from competitors

Formatting:
- Clear heading hierarchy
- Scannable sections
- Visual breaks every 300-400 words
- Custom images where valuable
```

### Voice Consistency (From DNA)
```
WitchcraftForBeginners:
✓ "Wise but warm" - experienced guide, not lecturer
✓ Patient and encouraging
✓ Acknowledges complexity without overwhelming
✓ Uses "we" and "your" (community feeling)
✗ NOT clinical or academic
✗ NOT preachy or dogmatic

MythicalArchives:
✓ Scholarly wonder - informed enthusiasm
✓ Rigorous but accessible
✓ Cross-cultural awareness
✓ Respectful of source cultures
✗ NOT dry academic
✗ NOT dismissive of beliefs
```

## Schema Implementation

### Primary Schema Types
```
Article (all posts)
HowTo (tutorials, rituals, processes)
FAQ (question sections)
Breadcrumb (navigation)
Organization/Person (about pages)
```

### Example HowTo Schema
```json
{
  "@type": "HowTo",
  "name": "How to Cast a Protection Spell",
  "description": "Step-by-step guide...",
  "totalTime": "PT30M",
  "tool": ["white candle", "salt", "protection oil"],
  "supply": ["journal", "matches"],
  "step": [
    {
      "@type": "HowToStep",
      "text": "Cleanse your space...",
      "image": "step1.jpg"
    }
  ]
}
```

## SEO Strategy

### Keyword Targeting
```
Head terms (pillar content):
- "witchcraft for beginners"
- "how to start witchcraft"
- "learn witchcraft"

Long-tail (cluster content):
- "simple protection spell for beginners"
- "what herbs for love spell"
- "waxing moon ritual ideas"

Question keywords (FAQ opportunity):
- "can anyone practice witchcraft"
- "is witchcraft dangerous"
- "what do I need to start witchcraft"
```

### Internal Linking Philosophy
```
Authority sites thrive on deep internal linking:
- Every post links to 3-5 related posts
- All clusters link to pillar
- Cross-reference constantly
- Update old posts to link to new
- Maintain a link map document
```

## Community Building

### Comment Strategy
```
Enable comments: Yes (builds community)
Moderation: Active (respond to comments)
Engagement: Reply within 24-48 hours
Tone: Warm, helpful, encouraging

This builds:
- E-E-A-T signals
- Return visitors
- Content ideas
- Community feeling
```

### Newsletter Strategy
```
Content: Beyond blog posts
- Exclusive tips
- Personal stories
- Community highlights
- Early access

Frequency: Weekly or bi-weekly
Voice: More personal than blog
Growth: Content upgrades, pop-ups (tasteful)
```

## Anti-Patterns for Authority Sites

```
NEVER:
✗ Thin, surface-level content
✗ Copying competitors verbatim
✗ Ignoring accuracy for speed
✗ Generic stock photos everywhere
✗ Pushy sales language
✗ Neglecting older content
✗ Keyword stuffing
✗ Ignoring reader questions/comments

ALWAYS:
✓ Go deeper than competitors
✓ Update content regularly
✓ Cite sources appropriately
✓ Use original images when possible
✓ Respond to community
✓ Maintain consistent voice
✓ Build genuine expertise signals
```
