## THE SEO STRUCTURAL ENGINEER'S TOOLKIT

### 1. Heading Hierarchy Architecture

Headings are structural, not stylistic.

**Heading Rules (Non-Negotiable):**

| Rule | Description |
|------|-------------|
| One H1 per page | The page's single topic declaration |
| Sequential order | H2 before H3, H3 before H4 |
| Semantic meaning | Headings describe content, not size |
| No skipping | Can't go H1 â†’ H3 (must have H2) |

**Heading Purpose Matrix:**

| Heading | Purpose | Example |
|---------|---------|---------|
| H1 | Page topic declaration | "Beginner's Guide to Witchcraft" |
| H2 | Major sections | "Essential Tools", "First Spells" |
| H3 | Subsections | "Candle Types", "Moon Phases" |
| H4 | Detail groups | "White Candles", "Waxing Moon" |
| H5-H6 | Rarely needed | Deeply nested content |

**Common Heading Mistakes:**

âŒ Using H2 because it "looks right" sized
âŒ Multiple H1s for stylistic purposes
âŒ Skipping levels (H1 â†’ H3)
âŒ Headings that don't describe content
âŒ H1 hidden for design reasons

### 2. Content Depth Strategy

Topical authority comes from depth, not breadth.

**Content Depth Ladder:**

| Level | Content Type | Purpose | Length |
|-------|--------------|---------|--------|
| **Pillar** | Comprehensive guide | Authority anchor | 3000-5000+ words |
| **Hub** | Topic overview | Connect related content | 1500-3000 words |
| **Spoke** | Specific subtopic | Detail and depth | 1000-2000 words |
| **Support** | Narrow question/answer | Long-tail capture | 500-1000 words |

**Authority Structure Pattern:**

```
                    [PILLAR]
                   /   |    \
              [HUB]  [HUB]  [HUB]
             /  |  \
        [SPOKE][SPOKE][SPOKE]
           |
       [SUPPORT]
```

**Empire Site Pillar Examples:**

| Site | Pillar Topic | Hubs | 
|------|--------------|------|
| WitchcraftForBeginners | "Complete Guide to Witchcraft" | Tools, Spells, Moon Magic, Sabbats |
| SmartHomeWizards | "Smart Home Ultimate Guide" | Hubs, Voice Assistants, Security |
| MythicalArchives | "World Mythology Encyclopedia" | Greek, Norse, Celtic, Egyptian |

### 3. Internal Linking Architecture

Internal links build authority flow.

**Link Flow Principles:**

| Principle | Implementation |
|-----------|----------------|
| **Pillar Pull** | All related content links to pillar |
| **Hub Distribution** | Hubs link to all spokes |
| **Contextual Relevance** | Links within relevant content |
| **Balanced Distribution** | No orphan pages |
| **Anchor Diversity** | Varied, natural anchor text |

**Internal Link Types:**

| Type | Purpose | Placement |
|------|---------|-----------|
| **Navigational** | Site structure | Menu, breadcrumbs |
| **Contextual** | Topic connection | Within content |
| **Related** | Continued reading | End of content |
| **Cross-pillar** | Authority bridges | Where topics intersect |

**Linking Density Guidelines:**

| Content Length | Internal Links |
|----------------|----------------|
| 500 words | 2-3 links |
| 1000 words | 4-6 links |
| 2000 words | 8-12 links |
| 3000+ words | 12-20 links |

### 4. Schema Markup Requirements

Schema tells search engines WHAT content is.

**Priority Schema Types:**

| Schema | When to Use | Impact |
|--------|-------------|--------|
| **Article** | All blog posts | Basic indexing |
| **HowTo** | Step-by-step content | Rich snippets |
| **FAQPage** | Q&A content | Featured snippets |
| **LocalBusiness** | Local-focused sites | Maps, local pack |
| **Product** | Product reviews | Rich results |
| **Recipe** | Recipe content | Rich cards |
| **Review** | Product reviews | Star ratings |
| **BreadcrumbList** | All pages | Navigation display |
| **Organization** | Homepage | Knowledge panel |
| **Person** | Author pages | Authority signals |

**Empire Site Schema Matrix:**

| Site | Primary Schema | Secondary |
|------|----------------|-----------|
| WitchcraftForBeginners | Article, HowTo, FAQPage | - |
| SmartHomeWizards | Article, HowTo, Product | Review |
| MythicalArchives | Article, FAQPage | - |
| Family-Flourish | Article, HowTo, FAQPage | Recipe |
| PulseGearReviews | Product, Review, Article | - |

**Schema Implementation Notes:**

- JSON-LD format (not microdata)
- One primary schema per page
- Can have multiple complementary schemas
- Must match visible content exactly
- Test with Google Rich Results Test

### 5. Semantic HTML Requirements

HTML structure matters for SEO.

**Required Semantic Elements:**

| Element | Purpose | SEO Impact |
|---------|---------|------------|
| `<header>` | Page/section header | Structure signal |
| `<nav>` | Navigation areas | Link identification |
| `<main>` | Primary content | Main content signal |
| `<article>` | Self-contained content | Content boundaries |
| `<section>` | Thematic grouping | Topic organization |
| `<aside>` | Tangential content | Sidebar distinction |
| `<footer>` | Page/section footer | Structure signal |

**Image SEO Requirements:**

| Attribute | Required | Format |
|-----------|----------|--------|
| `alt` | Always | Descriptive, keyword-aware |
| `title` | Optional | Additional context |
| `loading="lazy"` | Below fold | Performance |
| `width/height` | Always | CLS prevention |
| Filename | Before upload | descriptive-keyword.jpg |

**Link SEO Requirements:**

| Attribute | When | Purpose |
|-----------|------|---------|
| `rel="nofollow"` | Untrusted/paid links | PageRank control |
| `rel="sponsored"` | Paid/affiliate links | Disclosure |
| `rel="ugc"` | User-generated content | Source signal |
| `target="_blank"` | External links | UX + security (`rel="noopener"`) |

### 6. URL Structure

URLs are permanent SEO assets.

**URL Principles:**

| Principle | Good | Bad |
|-----------|------|-----|
| Descriptive | /moon-magic-guide/ | /post-123/ |
| Short | /candle-spells/ | /complete-guide-to-candle-spells-for-beginners/ |
| Lowercase | /witchcraft/ | /Witchcraft/ |
| Hyphenated | /full-moon-ritual/ | /full_moon_ritual/ |
| No parameters | /spells/ | /spells/?cat=1&id=42 |
| Flat preferred | /candle-spells/ | /spells/candle/types/white/ |

**Category URL Strategy:**

```
/[category]/[post-slug]/  (if category adds meaning)
/[post-slug]/             (if standalone)

Example:
/moon-magic/full-moon-ritual/     âœ… Category adds context
/full-moon-ritual/                 âœ… Also acceptable
/2024/01/15/full-moon-ritual/      âŒ Date adds no value
```

### 7. Core Web Vitals Considerations

Performance is an SEO factor.

**Critical Metrics:**

| Metric | Target | SEO Impact |
|--------|--------|------------|
| LCP (Largest Contentful Paint) | <2.5s | Ranking factor |
| FID (First Input Delay) | <100ms | Ranking factor |
| CLS (Cumulative Layout Shift) | <0.1 | Ranking factor |
| TTFB (Time to First Byte) | <600ms | Indirect |

**SEO-Impacting Design Decisions:**

| Design Element | SEO Consideration |
|----------------|-------------------|
| Hero images | Affects LCP |
| Web fonts | Affects LCP, CLS |
| Animations | Can affect CLS |
| Third-party scripts | Affects all metrics |
| Lazy loading | Affects LCP timing |
