## SECTION 1: CTA BUTTON TEXT

### Universal Forbidden CTAs

**NEVER use these on ANY empire site:**

```
FORBIDDEN (Generic):
- "Learn More"
- "Read More" 
- "Click Here"
- "Submit"
- "Get Started"
- "Sign Up"
- "Download"
- "Buy Now"
- "Order Now"
- "Shop Now"
- "Continue"
- "Next"
- "Go"
```

**Why forbidden:** Everyone uses these. They communicate nothing. They're forgettable.

### CTA Voice Principles

```
1. ACTION-ORIENTED but INVITING
   Bad: "Submit" (clinical)
   Bad: "Learn More" (vague)
   Good: "Start Your Journey" (inviting + specific)

2. OUTCOME-FOCUSED
   Bad: "Sign Up" (what they do)
   Good: "Join the Circle" (what they get)

3. BRAND-VOICED
   Bad: Generic applies anywhere
   Good: Only makes sense for THIS brand
```

## SECTION 2: FORM LABELS & PLACEHOLDERS

### Label Principles

```
1. CLEAR over clever
   Bad: "Drop us a line!" (for email field)
   Good: "Email Address"

2. HELPFUL context
   Bad: "Phone"
   Good: "Phone (optional - for order updates)"

3. NO redundancy
   Bad: Label "Email" + Placeholder "Enter your email"
   Good: Label "Email" + Placeholder "you@example.com"
```

### Form Field Patterns

**Email Fields:**
```
Label: "Email" or "Email Address"
Placeholder: "you@example.com"
Error: "Please enter a valid email address"
Success: (checkmark icon, no text needed)
```

**Name Fields:**
```
Label: "Name" or "Your Name"
Placeholder: (leave empty or "First Last")
Error: "Please enter your name"
NOT: "Full Name" (redundant "full")
```

**Comment/Message Fields:**
```
Label: "Message" or "Your Message"
Placeholder: Site-specific invitation
  - Witchcraft: "Share your thoughts with the coven..."
  - SmartHome: "Tell us about your setup..."
  - Family: "What's on your mind?"
NOT: "Enter your message here" (obvious)
```

**Search Fields:**
```
Placeholder (site-specific):
  - Witchcraft: "Search spells, rituals, guides..."
  - SmartHome: "Search devices, guides, reviews..."
  - Mythology: "Search myths, legends, deities..."
NOT: "Search" or "Type to search" (generic)
```

## SECTION 3: ERROR MESSAGES

### Error Message Principles

```
1. HELPFUL, not blaming
   Bad: "Invalid input"
   Bad: "You entered an invalid email"
   Good: "That doesn't look like an email address"

2. ACTIONABLE
   Bad: "Error occurred"
   Good: "Couldn't send. Check your connection and try again."

3. HUMAN tone
   Bad: "Form validation failed"
   Good: "Oops! We need a few more details."
```

### Error Message Library

**Form Errors:**
```
Required field empty:
- Generic: "This field is required"
- Better: "We need this to continue"
- Witchcraft: "This is needed for your journey"
- SmartHome: "We need this info"
- Family: "Help us help you‚Äîfill this in"

Invalid email:
- Generic: "Invalid email format"
- Better: "That doesn't look like an email address"
- All sites: "Hmm, that email doesn't look right"

Password too short:
- Generic: "Password must be 8 characters"
- Better: "Add a few more characters for security"

Passwords don't match:
- "Those passwords don't match‚Äîtry again"
```

**System Errors:**
```
Page not found (404):
- Generic: "Page not found"
- Witchcraft: "This path has vanished into the mist..."
- SmartHome: "This page seems to have disconnected"
- Mythology: "This scroll appears to be missing from the archives"
- Family: "Oops! This page wandered off"

Server error (500):
- Generic: "Something went wrong"
- All sites: "Something unexpected happened. We're looking into it."

Connection error:
- "Can't reach the server. Check your connection?"

Timeout:
- "This is taking longer than expected. Try refreshing?"
```

## SECTION 6: NAVIGATION LABELS

### Navigation Principles

```
1. CLEAR > Clever
   Bad: "The Vault" (for resources)
   Good: "Resources" or "Free Guides"

2. SCANNABLE
   One-two words preferred
   Action-oriented when possible

3. CONSISTENT across site
   Don't call it "Articles" in nav and "Blog" in footer
```

### Navigation Label Patterns

**Main Navigation (keep short):**
```
Good: "Guides" "Tools" "About" "Blog"
```

## SECTION 8: LOADING & WAITING STATES

### Loading State Principles

```
1. ACKNOWLEDGE the wait
   Something should happen immediately

2. COMMUNICATE progress (if possible)
   "Loading..." vs "Loading articles..." vs "Loading 3 of 10..."

3. BRAND the wait (if longer than 2s)
   Loading animations should match brand
```

### Loading Text Patterns

```
Generic loading:
  "Loading..." (acceptable but bland)

Site-specific loading:
  Witchcraft: "Consulting the spirits..." or "Gathering the ingredients..."
  SmartHome: "Connecting..." or "Syncing..."
  Mythology: "Searching the archives..."
  Family: "Finding what you need..."

Progress indication:
  "Loading articles (3 of 10)..."
  "Almost there..."
  "Just a moment more..."
```

## SECTION 9: CONFIRMATION DIALOGS

### Confirmation Principles

```
1. CLEAR action and consequence
   Bad: "Are you sure?" (sure about what?)
   Good: "Delete this comment? This can't be undone."

2. ACTION-LABELED buttons
   Bad: "OK" / "Cancel"
   Good: "Delete Comment" / "Keep It"

3. SAFE default highlighted
   The non-destructive option should be primary
```

### Confirmation Patterns

```
Delete confirmation:
  Title: "Delete this [item]?"
  Body: "This action cannot be undone."
  Buttons: [Cancel] [Delete] ‚Üê destructive is secondary/red

Unsaved changes:
  Title: "You have unsaved changes"
  Body: "Leave without saving?"
  Buttons: [Stay] [Leave]

Unsubscribe:
  Title: "We're sorry to see you go"
  Body: "You'll no longer receive our [frequency] emails."
  Buttons: [Stay Subscribed] [Unsubscribe]
```

## SECTION 10: ACCESSIBILITY TEXT

### Alt Text Principles

```
1. DESCRIBE the image purpose
   Not: "Image of woman"
   Yes: "Woman arranging candles on altar"

2. CONTEXT matters
   Decorative images: alt="" (empty, not missing)
   Informative images: Describe content
   Functional images: Describe function

3. NO "image of" or "picture of"
   Screen readers already announce it's an image
```

### Screen Reader Text

```
Skip link:
  "Skip to main content"

Read more (when hidden):
  <span class="sr-only">about [article title]</span>

Icon buttons:
  <button aria-label="Close menu">
  <button aria-label="Search">
  <button aria-label="Share on Twitter">

External links:
  <a>Resource Name <span class="sr-only">(opens in new tab)</span></a>
```

## SECTION 11: TIME & DATE FORMATTING

### Time Display Patterns

```
Recent (within 24h):
  "2 hours ago" / "Just now" / "Yesterday"

Within year:
  "March 15" (no year needed)

Older:
  "March 15, 2024"

NOT:
  "03/15/24" (ambiguous)
  "15 March 2024" (inconsistent)
```

### Reading Time

```
Format: "X min read"
Position: Near title/date

Example:
  "5 min read" NOT "5 minute read" or "5 mins"
```

## SECTION 12: NUMBERS & DATA

### Number Formatting

```
Large numbers:
  1,234 (with comma)
  1.2K (for 1,200+)
  1.2M (for millions)

Percentages:
  15% (no decimal unless meaningful)
  15.5% (when precision matters)

Currency:
  $19.99 (always two decimals for prices)
  $20 (acceptable for round numbers in content)
```
